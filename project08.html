Problématique

Pallier la disparition du VBA dans Outlook

Microsoft a progressivement supprimé la possibilité d'exécuter des macros VBA directement depuis Outlook. Le processus d'envoi d'e-mails automatisés aux clients devait donc être repensé en s'appuyant sur des outils encore disponibles dans l'écosystème Microsoft 365.

Contexte

Quand l'outil disparaît, l'automatisation survit

Le processus d'envoi d'e-mails clients reposait sur des macros VBA intégrées directement dans Outlook — une pratique répandue, que Microsoft a progressivement rendue impossible dans les nouvelles versions pour des raisons de sécurité.

Plutôt que de revenir à un envoi entièrement manuel, la solution a consisté à déplacer la logique là où le VBA fonctionne encore : dans Excel. La macro lit les données de la feuille, les envoie dans une liste SharePoint, et Power Automate prend le relais pour cibler le bon client et le bon interlocuteur sans aucune intervention supplémentaire.

Fonctionnement

De la cellule Excel à l'e-mail client

  1. Étape 01 — Excel
    Le fichier source contient toutes les données
    Le fichier Excel métier est le point de départ — il contient les informations client, les coordonnées du bon interlocuteur, et le contenu du message à envoyer, répartis dans des cellules structurées.
  2. Étape 02 — Macro VBA
    Un clic sur le bouton déclenche la macro
    Un bouton est intégré directement dans la feuille Excel et assigné à la macro VBA. Au clic, la macro lit les cellules concernées — client, e-mail, interlocuteur, contenu —, constitue un payload JSON et l'envoie vers la liste SharePoint via une requête HTTP POST. Une boîte de dialogue confirme le succès de l'opération.
  3. Étape 03 — SharePoint
    La liste reçoit les données et déclenche le flux
    La liste SharePoint enregistre l'entrée avec toutes les données nécessaires. L'ajout d'un nouvel élément dans la liste sert de déclencheur natif pour le flux Power Automate — aucun polling, aucun timer.
  4. Étape 04 — Power Automate
    Envoi ciblé au bon client et au bon interlocuteur
    Le flux récupère les données de la liste et envoie l'e-mail via Outlook au destinataire exact — client et interlocuteur identifiés à partir des informations extraites du fichier Excel. Aucune intervention manuelle n'est requise après le clic initial.
Fonctionnalités

Ce que la solution apporte

  • Macro VBA dans Excel — disponible et stable
    En déplaçant la logique d'Outlook vers Excel, la macro VBA reste pleinement fonctionnelle dans les versions actuelles de Microsoft 365 — sans dépendance à une fonctionnalité dépréciée.
  • Déclenchement au clic — zéro friction
    Un seul bouton dans le fichier Excel suffit à lancer l'intégralité du processus — lecture des données, envoi dans SharePoint, déclenchement du flux et envoi de l'e-mail client.
  • SharePoint comme pont entre Excel et Power Automate
    La liste SharePoint joue le rôle de couche d'intégration : elle reçoit les données du VBA et déclenche nativement le flux Power Automate dès l'ajout d'une nouvelle entrée, sans configuration complexe.
  • Envoi ciblé — bon client, bon interlocuteur
    Power Automate utilise les données extraites du fichier pour adresser l'e-mail précisément au destinataire voulu — sans risque d'erreur de destinataire lié à une saisie manuelle.
  • Architecture découplée et maintenable
    Chaque couche (VBA, SharePoint, Power Automate) est indépendante — on peut faire évoluer le template e-mail dans Power Automate sans toucher à la macro, et vice versa.

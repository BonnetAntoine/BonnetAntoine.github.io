Logique métier

Une condition qui change le document

Selon le profil du technicien concerné, le document généré n'est pas identique. Deux conditions peuvent s'appliquer indépendamment et modifier une ligne spécifique du PDF — celle relative à l'accord du salarié.

Travail de nuit
Condition 1
La modification implique des heures de nuit
Dès que la plage horaire renseignée par le responsable couvre des heures de nuit, la condition se déclenche automatiquement via la logique PowerFX.
→ La ligne "Accord du salarié" est ajoutée dans le document PDF avec un espace de signature — l'accord explicite du technicien est requis réglementairement.
Délégué du personnel
Condition 2
Le technicien concerné est délégué du personnel
Si le responsable identifie le technicien comme délégué du personnel dans le formulaire, cette case coche active une règle spécifique dans le générateur de document.
→ La ligne "Accord du salarié" est insérée dans le PDF — le statut de représentant du personnel impose une validation préalable de l'intéressé.
Document généré

Anatomie du PDF produit

L'application compose le document à la volée à partir des données saisies dans les deux formulaires. Le résultat est un PDF conforme, prêt à être archivé et distribué.

Contexte

Formaliser une obligation légale

Toute modification des horaires de travail d'un technicien doit faire l'objet d'un document officiel transmis aux personnes concernées dans les délais réglementaires. Avant cette application, ce document était saisi manuellement dans Word, imprimé, signé puis scanné — un processus long, source d'erreurs et difficile à tracer.

L'objectif était de digitaliser entièrement la création de ce document depuis l'application : saisie guidée, génération automatique du PDF conforme, archivage en base et envoi instantané par e-mail aux personnes concernées, le tout sans quitter Power Apps.

Fonctionnement

Le flux de bout en bout

  1. Formulaire 01
    Informations du document
    Le responsable renseigne les données contextuelles : techniciens concernés, site d'intervention, nom du client, date de création, motif de la modification. Il coche également si le technicien est délégué du personnel — ce qui active la condition sur la ligne d'accord.
  2. Formulaire 02
    Plages de dates et d'horaires
    Le responsable sélectionne la période concernée et renseigne pour chaque jour les horaires initiaux et les nouveaux horaires. La saisie d'heures de nuit déclenche automatiquement la condition correspondante dans le générateur de document.
  3. Génération
    Création du document PDF
    L'application compose le PDF à partir de toutes les données saisies. La logique conditionnelle PowerFX insère ou omet la ligne "Accord du salarié" selon les conditions actives. Le document généré est conforme, complet et prêt à être signé.
  4. Archivage
    Enregistrement en base SharePoint
    Le PDF généré est stocké dans SharePoint avec l'ensemble des données du formulaire : un historique complet des notes de modification est ainsi constitué, consultable à tout moment depuis l'application.
  5. Envoi automatique
    E-mail aux personnes concernées
    Un flux Power Automate récupère le document archivé et envoie un e-mail avec un template HTML structuré — récapitulatif des dates et heures modifiées — aux techniciens et aux parties concernées, avec le PDF en pièce jointe.
Fonctionnalités

Ce que l'application apporte

  • Saisie guidée en deux formulaires
    Un premier formulaire pour les informations contextuelles, un second pour les plages horaires — la logique PowerFX assure la cohérence des données avant génération.
  • Logique conditionnelle sur le document
    Travail de nuit ou délégué du personnel : deux conditions détectées automatiquement qui insèrent la ligne "Accord du salarié" dans le PDF généré — sans action supplémentaire du responsable.
  • Génération PDF à la volée
    Le document est composé dynamiquement depuis l'application — conforme, complet, avec les bonnes sections selon le contexte. Aucune mise en page manuelle n'est requise.
  • Archivage automatique en SharePoint
    Le PDF et toutes les données saisies sont stockés dans une liste SharePoint — un historique traçable et consultable de toutes les notes de modification émises.
  • Envoi e-mail automatisé avec pièce jointe
    Power Automate envoie le PDF en pièce jointe avec un template HTML Outlook récapitulant les dates et heures modifiées — directement aux techniciens et aux parties concernées.
  • Usage bureau exclusif
    Application conçue pour le personnel de bureau et les responsables — interface PC optimisée pour la saisie et la gestion des documents générés.
